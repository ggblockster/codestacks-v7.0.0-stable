File I/O Reading

CodeStacks - Python

Abstract

In Python, you can write text to files, as well as read data from those files. On top of that, you can connect multiple files together and use values and functions between them so that your Python files don't become too long.

Overview

In Python, you can access files with the built-in open() function. But in order for that to work, you must specify the path of the file, and place said path, formatted as a string.

Assuming that file.txt exists in the same folder as the Python file the below snippet is running in, you can essentially get Python to read the file. 

open("file.txt")

The open() function takes two parameters. The file path, and the mode.

Modes

Since we can access files inside of Python, let's take a look at everything you can do with it.

By default, Python will assume that the mode that you have selected is read-text. Let's look at all the mode types that you can use.

The first half must be r, a, w, and x, and the second half must be a t or a b.

Name Code Point Description
Read r Open a file to read. If it doesn't exist, then it returns an error.
Append a Opens a file to append. If it doesn't exist, then it returns an error.
Write w Opens a file to write. If it doesn't exist, then it returns an error.
Create x Creates a file with the specified name. If it already exists, then it returns an error.
Text t Sets the input type of the mode to text-only.
Binary b Sets the input type of the mode to binary-only, which can render images as well.

For selecting the mode, you need to make sure that t or b, whichever type you choose, that must be on the end of that string. Let's take a look at some examples on formatting.

rt means "read text". Similarly, xt means "create text file". This is how you format these modes.

Reading Files

Now that you can open files in order to access them, let's look at how to read the data inside those files. 

# Opening a file:
f = open("demo.txt", "rt")

# Reading the file
r = f.read()
print(r)
f.close()

Assuming demo.txt is this: 

Hello!
This is demo.txt.
This is an example file for testing purposes.

Output: 

Hello!
This is demo.txt.
This is an example file for testing purposes.

Make sure that if you open a file, you can read it, but, if you make any changes, you cannot save to the file until you call close(). Another thing to note is that these functions are being called from variables. If close() isn't called, then you will end up with a MemoryError after some repeats.


The WITH Command


If closing the file itself is too much of a hassle, you can use the with command. In this example, we will reuse demo.txt. 

# Using WITH:
with open("demo.txt", "rt") as f:
    print(f.read())

By using with, you don't have to worry about closing the file manually.



Reading Part of Files

What if you wanted to only use part of a file? You can actually do that. Notice the use of brackets in read()? Well, you can add a parameter to that. A number. And whatever that number is, will return that many characters of that file. So f.read(5) will return Hello if f refers to demo.txt.

There's also readline(). Leaving the brackets blank will automatically return one line of the connected file. If you want to return only two full lines of f, which is demo.txt, you can do f.readline(2).

Writing to Files

There are two possible combinations for the mode parameter in open(). And those are at and wt. In plain text, that becomes appending to the end of the file, a, and overwriting the existing content in the file, w.

Now that you've established the file you wish to edit, attach write() to the defined variable, and add the string you wish to add/overwrite. Here is an example. 

# Example:
with open("demo.txt", "at") as f:
    f.write("Now the file has more content!")

with open("demo.txt", "rt") as f:
    print(f.read())

That was just an example of how to append text to the file. Because of us calling with, the file automatically saves and closes, so when we next, call f.read(), it now contains Now the file has more content! added to the end. You can also do the same thing with wt, but that will just overwrite any content in the file that was already there.

Creating New Files

To create new files, you can apply a single character as the second parameter for open(). To create a new file, that character should be a, w, or x. The first parameter should be the name of the new file. If the file you attempt to create already exists, Python will return an error. As a result of successful creation, you will now have a blank, empty file of the type you specify. 

# Example
f = ("newfile.txt", "x")

Result: A new, empty file by the name of newfile.txt is created.

Deleting Files

In order to delete files, you need to import the os module by calling its os.remove() command. 

# Example:
import os

os.remove("demo.txt")

However, you can't automatically assume that the file will always be there. That's where you can implement a try...except block. Let's try the same thing again, this time, using try...except : 

# Example
import os

try:
    os.remove("demo.txt")
except:
    print("demo.txt could not be removed.")

In Python, you can also delete entire folders. The os module has a command for this, and that is os.rmdir("myfolder") 

# Removing a folder:
import os

os.rmdir("myfolder")

warning NOTE:
Only empty folders can be deleted using os.rmdir .
