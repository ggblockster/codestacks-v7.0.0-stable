Functions

CodeStacks - Python

Abstract

Functions in Python are really useful, just like loops. Instead of writing certain segments of the same code repeatedly in a row, what if you wanted to re-use the same code in different parts of your code? Well, the easy solution is functions. Theres a way to define them and to use them, and you won't ever need to write a specific segment of code multiple times when you can just write it once and attach it to a function.

Overview

Functions is Python are structured in a specific way. You need to have the defining keyword, def, the name of the function, and then that's directly followed by brackets, opened and closed, then a colon. What goes into the brackets? Well, those are called parameters. Parameters are values you can input into the function when you call on it, and you can then treat those as variables within the function.

An important part of using functions (with syntax) is having an indent to put your code in, otherwise it won't count as part of the function. Here's a basic example of a function in Python: 

def add(num1, num2):
    result = num1 + num2
    return result

print(add(3,5)) # Outputs 8.

Returning Values

Now that we know how to use parameters and functions, how do we output values based on those parameters? Well the solution is the return keyword. Anything after the keyword (it has to follow Python syntax to avoid errors), and also not a comment, will be outputted in place of the original calling of the function.

Let's look at a similar example to the one before, but looking at the return keyword. 

# Example:
def multiply(num1, num2):
    result = num1 * num2
    return result

print(multiply(3,5))

This time, we multiply the parameters. But look — notice the return keyword? The item after the keyword is the name of a variable, the result variable. The variable holds the product of the two parameters.

Function Names

There are a couple of rules toward naming functions. They must start with a letter or an underscore.

  • They can only start with letters or underscores.
  • They can contain letters, numbers, or underscores.
  • They are case-sensitive. That means myfunction() and myFunction() are considered different.

Passing

If there is a case in which you need to create a function, but you don't need to use it yet, you can use the pass statement. You can place it inside empty functions, effectively safelocking them from turning into syntax errors. 

# Empty Function Example:
def myFunction():
    pass

myFunction() # Returns nothing.

The pass function is particularly useful when you'd prefer to build the inner structure before implementing the function itself.
