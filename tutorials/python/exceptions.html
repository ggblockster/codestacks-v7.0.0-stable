Exceptions

CodeStacks - Python

Abstract

Exception handling, better known as the try..except block, can be used to handle possible errors in a clean way without having to see a syntax error stopping the rest of your code from running.

Overview

The try...except block in Python has four parts to it. The try section, where Python tries to execute the code, the except section, which is executed when there is an error, the else block, which runs when there are no errors, and the finally block, which runs regardless.

When creating a try.except block, you can leave out any part except the try and the except. You can have multiple exceptions, but only one try, and a maximum of one for else or finally. 

# Example
try:
  print(x)
except:
  print("An error occurred")

Because we have a try...except block, we can print a message to the output window if Python finds an error, as x is not defined. 

print(x)

The above code snippet will raise an error immediately because we never defined x.

Parts of a try..except Block


Try

The try section allows you to run code assuming that it won't raise an error. Any code inside it is safe because of exception handling. 

# Example:
try:
    print(x)

And the output: 

Syntax Error

There is a syntax error because the block doesn't have an except block to pair with it.


Except


These are essential when you create a try...except block. The except block contains the code that you run if there was an error in running the code in the try block. 

try:
    print(x)
except:
    print("There was an error.")

Else


The else block exists to run code if there are no errors present for when the try block was able to succesfully run. 

try:
    print(x)
except:
    print("There was an error.")
else:
    print("There were no errors.")

Finally


The finally block in the try...except block will always run, regardless of whether you have an error or you don't. 

x = 5

try:
    print(x)
except:
    print("There was an error.")
else:
    print("There were no errors.")
finally:
    print(f"x is equal to {x}.")

Multiple Exceptions

Apart from having just one exception, you can cipher errors into specific cases. If the type isn't specified, then Python will assume that the block will run on any type of error. For example, if you want to run some code if there's a NameError, you can! 

# Exceptions:
x = 5

try:
    print(y)
except NameError:
    print("NameError")
except:
    print("There was an error.")

Error Types

Since we covered that you can specify under which error that an except block can run, let's look at the types of errors that you can choose from:

Raising Errors

If you want to define what you consider an error in your code, you can use the raise keyword, then add the name of an error, and Python will close the program respecting the error. 

x = -1

if x < 0:
    raise ValueError("No numbers below 0.")
